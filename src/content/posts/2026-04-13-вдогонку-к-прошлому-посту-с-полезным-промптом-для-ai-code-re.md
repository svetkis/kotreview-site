И тебе приходится делать это за них, поскольку никто в 2026 не хочет вычитывать глазами все ошибки, которые быстрее поймает LLM. Тем более что после этого сил на архитектуру может и не остаться. У меня есть лайфхаки, как получить данные для code review максимально просто.

**1. Подходит для простых MR, можно скачать diff прямо из GitLab**

- Заходим в GitLab на страницу, где отображаются изменения.
- В адресной строке браузера окончание строки .diffs меняем на .diff.
- Вуаля, у нас прямо в браузере загрузился git diff (могут быть нюансы с кодировкой для русских символов).

**2. Если MR сложный, и важен также код, который в diff не виден** есть способ скачать MR как ветку прямо в GitLab. Для этого не нужны права на ветку разработчика. Нужно, если хотите комплексно посмотреть MR, так как не всегда достаточно только diff'a.

__Как скачать ветку MR для ревью прямо из GitLab:__

- Смотрим номер MR.
- Подставляем номер (например, 1223) в команды:
    git fetch origin merge-requests/1223/head:mr-1223
    git checkout mr-1223
