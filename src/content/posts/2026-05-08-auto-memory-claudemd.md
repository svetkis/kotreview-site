**Auto Memory ≠ ****CLAUDE.md**

Сергей (@naguala) в комментах написал, что Claude Code сам раскидывает память по файлам и в CLAUDE.md не пишет. Вопрос настолько интересный, что отвечу постом.

В целом это так: Auto Memory появилась в конце февраля 2026, и агент действительно сам ведёт заметки о проекте.

**Но как говорится есть нюанс.**

**Auto Memory плоская.** Она привязана к корню git-репозитория. Находишься ты в src/Ordering/Application/, src/Payment/Infrastructure/ или src/Catalog/Web/ — агент всё равно тянет одну и ту же память. Контекста папки он не понимает.

**CLAUDE.md****, наоборот, иерархический.** Кладешь его в src/Ordering/Domain/ — получаешь контекст именно для доменной модели. Кладешь в src/Payment/Infrastructure/ — получаешь совсем другой контекст (EF Core-конфигурации, Outbox, resilience и т.д.).

На маленьком проекте разницы почти нет. 

**На проекте в 30–50+ модулей это две совершенно разные вселенные.**

Auto Memory хорошо запоминает практические мелочи: что тесты запускаются через dotnet test --no-build --filter, что ты предпочитаешь record вместо классов в DTO.

А CLAUDE.md кодирует осознанные архитектурные решения: границы bounded contexts, принятые паттерны, правила расположения файлов.

**По сути это два разных инструмента:**

**Auto Memory** — заметки агента для самого себя.
**CLAUDE.md** — архитектурный контекст, который проектирует разработчик.

На больших проектах заменить одно другим не получится.
