Знакомая картина? Claude Code на проекте больше пары тысяч строк ведёт себя так:

- читает файл
- открывает соседний, читает его тоже
- делает regex-replace
- ломает ссылку где-то на другом конце кодовой базы
- читает, что сломал
- исправляет сломанное
- usage тем временем ускоренно выжирается

## Решение

Serena MCP подключает полноценный Language Server Protocol. У агента появляются инструменты: `find_symbol`, `find_referencing_symbols`, `insert_after_symbol`, `get_symbols_overview`.

Агент перестаёт читать файлы целиком и начинает работать точечно: «дай мне определение этой функции», «найди все её использования», «вставь код после такого-то символа».

## Что это даёт

- Экономим токены.
- Точность правок выше, чем у regex-replace.
- Рефакторинги становятся безопасными.

Это must-have для разработки с агентами на больших проектах.
